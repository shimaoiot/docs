    # 设备OTA固件升级成功失败通知

    发起方：服务端

    协议：MQTT

    method：ota-notify-record

    上行 Topic：/topic/pub/device/{AppKey}/{clientId}

    下行 Topic：/topic/sub/device/{AppKey}/{clientId}

    # header 信息

    header 信息包含所有请求必须的通用信息

    # 示例

    {
  "header": {
    "traceId": "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx",
    "payloadVersion": 1,
    "brand": "KT",
    "timestamp": 1234567890123,
    "method": "ota-notify-record"
  }
}

    # 请求参数说明

    参数名 参数类型 参数说明 是否必填
    otaFirmwareId Long 设备固件维护id（即包拉取的id） 是
    deviceCode String 设备 code 是
    currTime long 当前时间错 是
    deviceAppKey String 升级包下载链接 是
    errorMsg String 错误信息 status 1-失败时必填 否
    status Int 状态，状态 0-成功 失败 10001-appkey 错误 10002-deviceCode 错误 11000-未知错误 11001-version 不是字符串 11002-版本解析失败 11003-版本比较失败 11004-新程序版本匹配错误 11005-下载文件失败 11006-文件解析失败 是

    协议体

    {
  "header": {
    "method": "ota-notify-record",
    "brand": "haiman",
    "payloadVersion": 1,
    "timestamp": 1605601573866,
    "traceId": "04066763d1bd451da4c900192d342354"
  },
  "payload": {
    "otaFirmwareId": 733,
    "deviceCode": "65ead0c8f1f9706dc295c5bd54a16303",
    "currTime": 1234567890,
    "deviceAppKey": "65ead0c8f1f9706dc295c5bd54a16303",
    "errorMsg": "因为网络原因失败",
    "status": 1
  }
}
